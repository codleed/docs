## Overview

The Boards API returns all educational boards (e.g., Federal Board, Punjab Board, Sindh Board) configured in the system. Boards are the top-level curriculum grouping — each board contains multiple classes.

<Warning>
This router is defined in `backend/src/routes/boards.ts` but is **not currently mounted** in `server.ts`. The boards data is instead served through the admin content endpoints and the learn routes.
</Warning>

---

## List boards

Returns all boards ordered alphabetically by name.

```bash
GET /api/boards
```

**Auth**: None (public endpoint)

### Example

<CodeGroup>

```bash cURL
curl http://localhost:3001/api/boards
```

</CodeGroup>

### Response

**Status**: `200 OK`

```json
{
  "boards": [
    {
      "id": 1,
      "name": "Federal Board",
      "slug": "federal"
    },
    {
      "id": 2,
      "name": "Punjab Board",
      "slug": "punjab"
    }
  ]
}
```

<ResponseField name="boards" type="array" required>
  Array of board objects.

  <Expandable title="board object">
    <ResponseField name="id" type="number" required>
      Unique board identifier.
    </ResponseField>

    <ResponseField name="name" type="string" required>
      Display name of the board (e.g., `"CBSE"`).
    </ResponseField>

    <ResponseField name="slug" type="string" required>
      URL-safe slug used in curriculum routes (e.g., `"cbse"`).
    </ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
|--------|-----------|
| `500` | Database error — failed to fetch boards |
