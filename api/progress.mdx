## Overview

The Progress API tracks each student's learning journey through progress events. Events are recorded when students visit chapters, view exercises, complete flashcards, or submit quizzes.

---

## Record progress event

Submit a learning activity event to update the student's progress.

```bash
POST /api/progress/events
```

**Auth**: Student

### Request body

The request body uses a discriminated union based on `eventType`:

<Tabs>
  <Tab title="chapter_visit">
    <ResponseField name="eventType" type='"chapter_visit"' required />
    <ResponseField name="chapterId" type="number" required>
      The chapter ID that was visited.
    </ResponseField>
  </Tab>
  
  <Tab title="exercise_view">
    <ResponseField name="eventType" type='"exercise_view"' required />
    <ResponseField name="chapterId" type="number" required>
      The chapter ID whose exercise was viewed.
    </ResponseField>
  </Tab>
  
  <Tab title="flashcard_complete">
    <ResponseField name="eventType" type='"flashcard_complete"' required />
    <ResponseField name="chapterId" type="number" required>
      The chapter ID whose flashcards were completed.
    </ResponseField>
  </Tab>
  
  <Tab title="quiz_submit">
    <ResponseField name="eventType" type='"quiz_submit"' required />
    <ResponseField name="chapterId" type="number" required>
      The chapter ID of the submitted quiz.
    </ResponseField>
    <ResponseField name="score" type="number" required>
      The quiz score (non-negative integer).
    </ResponseField>
  </Tab>
</Tabs>

### Example

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3001/api/progress/events \
  -H "Content-Type: application/json" \
  -H "Cookie: better-auth.session_token=..." \
  -d '{
    "eventType": "chapter_visit",
    "chapterId": 5
  }'
```

```json Quiz submit example
{
  "eventType": "quiz_submit",
  "chapterId": 5,
  "score": 8
}
```

</CodeGroup>

### Response

```json
{
  "recorded": true,
  "eventType": "chapter_visit",
  "chapterId": 5
}
```

### Error responses

| Status | Condition |
|--------|-----------|
| `400` | Invalid progress event payload — missing fields or wrong types |
| `500` | Internal error recording progress |

## Streak Wager

Place a wager to protect your streak. If you fail, you lose the wagered XP instead of your streak.

```bash
POST /api/progress/streak-wager
```

**Auth**: Student

### Request body

| Field | Type | Description |
|-------|------|-------------|
| `wagerAmount` | number | XP amount to wager (minimum 10, must have sufficient XP) |

### Response

```json
{
  "success": true,
  "data": {
    "wagerId": 1,
    "wagerAmount": 50,
    "status": "active",
    "expiresAt": "2026-04-11T00:00:00.000Z"
  }
}
```

### Notes
- If streak breaks, user loses wagered XP instead of streak
- If streak continues, no penalty
- Only one active wager at a time

## Recover Streak Wager

Attempt to recover a broken streak wager by answering a recovery question.

```bash
POST /api/progress/streak-wager/recover
```

**Auth**: Student

### Request body

| Field | Type | Description |
|-------|------|-------------|
| `wagerId` | number | The wager ID to recover |
| `answer` | string | Answer to recovery question |

### Response

```json
{
  "success": true,
  "data": {
    "recovered": true,
    "wagerAmount": 50,
    "streak": 12
  }
}
```

### Notes
- Recovery is time-limited
- If successful, streak is restored and wager is returned

## Complete Today's Focus

Mark today's focus task as complete and claim the associated XP reward.

```bash
POST /api/progress/todays-focus/complete
```

**Auth**: Student

### Response

```json
{
  "success": true,
  "data": {
    "completed": true,
    "xpEarned": 5,
    "currentStreak": 15,
    "nextFocus": {
      "type": "chapter_visit",
      "chapterId": 5,
      "xpReward": 5
    }
  }
}
```

### Notes
- Called when user completes their daily focus task
- Focus tasks are dynamically generated based on learning goals
- XP reward varies by task type

## Get subject dashboard

Retrieve detailed progress for a specific subject, including chapter-by-chapter breakdown.

```bash
GET /api/progress/dashboard/:boardSlug/:grade/:subjectSlug
```

**Auth**: Student

### Path parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `boardSlug` | string | Board slug — lowercase alphanumeric with hyphens |
| `grade` | `"9"` \| `"10"` | Grade |
| `subjectSlug` | string | Subject slug — lowercase alphanumeric with hyphens |
