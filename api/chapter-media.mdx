## Overview

The Chapter Media API allows admins to upload images associated with chapter summaries. Media files are stored in MinIO and linked to the chapter through the `chapter_summary_media` table. The response includes a Markdown image tag ready to embed in the chapter summary.

---

## Upload chapter summary media

Upload an image for a chapter summary. Requires admin privileges.

```bash
POST /api/admin/content/chapters/:chapterId/summary-media
```

**Auth**: Admin (requires valid session with `role: "admin"`)

### Path parameters

<ResponseField name="chapterId" type="number" required>
  The unique identifier of the chapter to attach media to. Must be a positive integer.
</ResponseField>

### Request

Send as **multipart/form-data** with a single file field.

<ResponseField name="media" type="file" required>
  The image file to upload. Field name must be `media`.

  **Supported formats**: JPEG, PNG, WEBP, GIF

  **Maximum size**: 10 MB
</ResponseField>

### Example

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3001/api/admin/content/chapters/42/summary-media \
  -H "Cookie: better-auth.session_token=..." \
  -F "media=@/path/to/diagram.png"
```

</CodeGroup>

### Response

**Status**: `201 Created`

```json
{
  "asset": {
    "id": 15,
    "chapterId": 42,
    "objectUrl": "https://storage.example.com/learningopk/chapters/42/user-abc123/diagram.png",
    "mimeType": "image/png",
    "fileSize": 245760,
    "createdAt": "2026-03-30T12:00:00.000Z"
  },
  "markdown": "![Chapter summary media](https://storage.example.com/learningopk/chapters/42/user-abc123/diagram.png)"
}
```

<ResponseField name="asset" type="object" required>
  The persisted media asset record.

  <Expandable title="asset object">
    <ResponseField name="id" type="number" required>
      Unique media asset identifier.
    </ResponseField>

    <ResponseField name="chapterId" type="number" required>
      ID of the chapter this media belongs to.
    </ResponseField>

    <ResponseField name="objectUrl" type="string" required>
      Public URL of the uploaded media file.
    </ResponseField>

    <ResponseField name="mimeType" type="string" required>
      MIME type of the uploaded file (e.g., `"image/png"`).
    </ResponseField>

    <ResponseField name="fileSize" type="number" required>
      Size of the file in bytes.
    </ResponseField>

    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp of when the media was uploaded.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="markdown" type="string" required>
  A pre-formatted Markdown image tag that can be directly inserted into the chapter summary editor.
</ResponseField>

### Error responses

| Status | Condition |
|--------|-----------|
| `400` | Invalid `chapterId` — must be a positive integer |
| `400` | Missing `media` file in the multipart payload |
| `400` | Unsupported image type — only JPEG, PNG, WEBP, and GIF are allowed |
| `401` | No valid session — user is not authenticated |
| `403` | User does not have admin role |
| `404` | Chapter with the given ID does not exist |
| `413` | File exceeds the 10 MB size limit |
| `500` | Server error — failed to upload to MinIO or persist the media record |

<Warning>
If the upload succeeds but the database insert fails, the newly uploaded object is automatically deleted to prevent orphaned files in object storage.
</Warning>

<Note>
Each upload creates a new media record. Multiple images can be attached to the same chapter. The `markdown` field in the response makes it easy to embed images into the chapter summary content.
</Note>

## Get presigned upload URL

Get a presigned PUT URL for direct browser-to-MinIO uploads.

```bash
POST /api/admin/content/chapters/:chapterId/presigned-upload
```

**Auth**: Admin

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `fileName` | string | Yes | Original file name |
| `mimeType` | string | Yes | MIME type (image/jpeg, image/png, image/webp, image/gif) |
| `fileSize` | number | Yes | File size in bytes (max 10 MB) |

### Response

```json
{
  "uploadUrl": "https://minio:9000/learningo-media/...",
  "objectKey": "chapters/42/user-abc123/diagram.png"
}
```

## Confirm presigned upload

Confirm a presigned upload and create the database record.

```bash
POST /api/admin/content/chapters/:chapterId/media/confirm
```

**Auth**: Admin

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `objectKey` | string | Yes | The object key returned from presigned upload |
| `mimeType` | string | Yes | MIME type of the uploaded file |
| `fileSize` | number | Yes | File size in bytes |

### Response

```json
{
  "asset": {
    "id": 15,
    "chapterId": 42,
    "objectUrl": "https://storage.example.com/learningopk/chapters/42/user-abc123/diagram.png",
    "mimeType": "image/png",
    "fileSize": 245760,
    "createdAt": "2026-03-30T12:00:00.000Z"
  }
}
```

## List chapter media

List all media files for a chapter.

```bash
GET /api/admin/content/chapters/:chapterId/media
```

**Auth**: Admin

### Response

```json
{
  "media": [
    {
      "id": 15,
      "chapterId": 42,
      "objectUrl": "https://storage.example.com/learningopk/chapters/42/user-abc123/diagram.png",
      "mimeType": "image/png",
      "fileSize": 245760,
      "createdAt": "2026-03-30T12:00:00.000Z"
    }
  ]
}
```

## Delete chapter media

Delete a media file from both MinIO and the database.

```bash
DELETE /api/admin/content/chapters/:chapterId/media/:mediaId
```

**Auth**: Admin

### Response

```json
{
  "success": true
}
```

### Error responses

| Status | Condition |
|--------|-----------|
| `404` | Media file not found |
| `500` | Failed to delete from MinIO or database |
