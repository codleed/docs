## Shared package

The monorepo includes a `@learningopk/shared` workspace package used by both frontend and backend for shared TypeScript types and constants.

```json
// Both packages depend on it
"@learningopk/shared": "workspace:*"
```

## Build and run commands

<Tabs>
  <Tab title="Development">
    ```bash
    # Start everything
    pnpm dev:all

    # Or individually
    pnpm dev:frontend   # Next.js on port 3000
    pnpm dev:backend    # Express on port 3001
    ```
  </Tab>
  <Tab title="Database">
    ```bash
    pnpm docker:up      # Start PostgreSQL + Redis + MinIO
    pnpm db:generate    # Generate Drizzle migrations
    pnpm db:migrate     # Run migrations
    pnpm db:seed        # Seed content from PDFs
    pnpm db:studio      # Open Drizzle Studio
    ```
  </Tab>
  <Tab title="Testing">
    ```bash
    # Backend
    pnpm --filter backend test           # All tests
    pnpm --filter backend test:unit      # Unit tests
    pnpm --filter backend test:integration  # Integration tests

    # Frontend
    pnpm --filter frontend test:e2e      # Playwright E2E
    pnpm --filter frontend test:e2e:smoke  # Smoke tests only
    ```
  </Tab>
  <Tab title="Verification">
    ```bash
    pnpm verify:services    # Health check all services
    pnpm --filter backend ai:verify     # Test AI chat
    pnpm --filter backend quiz:verify   # Test quiz submission
    pnpm --filter backend auth:verify   # Test authentication
    ```
  </Tab>
</Tabs>

## Why these choices

- **No vendor lock-in** — everything runs locally via Docker; no cloud services required for development
- **Free tier AI** — Mistral's free tier provides 1B tokens/month, sufficient for investor demos
- **Type safety end-to-end** — TypeScript strict mode across both frontend and backend, with shared types via `@learningopk/shared`
- **pnpm workspaces** — faster installs than npm, native monorepo support without Turborepo complexity
